## 💾 قاعدة البيانات

### المجموعات المستخدمة:

**1. admins** - بيانات المسؤولين
```
admins/{adminId}
├── email: string
├── fullName: string
├── role: super_admin | admin
├── phoneNumber: string
├── isActive: boolean
└── permissions: array[string]
```

**2. doctor_requests** - طلبات الأطباء
```
doctor_requests/{requestId}
├── doctorId: string
├── fullName: string
├── specialty: string
├── status: pending | approved | rejected
├── documents: { license, degree, etc }
├── createdAt: timestamp
├── reviewedBy: string (adminId)
└── rejectionReason: string (if applicable)
```

**3. admin_logs** - سجل الإجراءات
```
admin_logs/{logId}
├── adminId: string
├── action: string (approve, reject, etc)
├── doctorId: string
├── timestamp: timestamp
└── details: map
```

```
[✅ شهادة التخرج]

[✅ الموافقة] [❌ الرفض]
```

## 🔄 سير العمل

```
طبيب يتقدم بطلب
        ↓
بيانات تحفظ (pending)
        ↓
مسؤول يدخل /admin
        ↓
يشاهد الطلبات المعلقة
        ↓
يفتح تفاصيل الطلب
        ↓
يراجع البيانات والوثائق
        ↓
      ├─ ✅ موافقة → verified ✓
      │
      └─ ❌ رفض → مع السبب
        ↓
يُسجل الإجراء في السجل
```
